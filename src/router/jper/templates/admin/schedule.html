This page shows Router's batch process schedule - either the initial configuration or the active schedule obtained from the database.

Time values are always shown as GMT (i.e. not adjusted for summer time).

Processes are synchronised across servers and application programs. They may run with a regular frequency and/or may be triggered (if they have a label) by the completion of another process. Processes will wait until a conflicting process finishes running.

You can manually trigger a process to run immediately using the 'run' button in the last column.

The past history of process execution can be seen on the Batch Process Metrics page.

Status values

StatusDescription
TriggeredProcess is ready to start, having been triggered by completion of another process.
PendingProcess is scheduled to run when its next start time is reached.
WaitingProcess would have started already, but is waiting until a conflicting process finishes running.
RunningProcess is currently running.
FinishedProcess has run and is not currently scheduled to run again, but may be triggered in the future.
EndedProcess cannot run as its 'latest run time' is in the past.
(none)Process not scheduled, but can be triggered.
