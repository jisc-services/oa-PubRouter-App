Manage matching

Your repository will receive notifications that have been matched to your matching parameters (view in new window) and which satisfy your maximum age criteria (if set).

Maximum age (optional)

As well as notifications about newly-published articles, some Publications Router content providers, such as Crossref, occasionally provide updated metadata for articles published some time ago (possibly several years ago).

If you wish to filter your data feed to exclude notifications about older articles you may set a maximum age (in years) since article publication. This is used to calculate an earliest publication date by subtracting that number of years from the current date each time the matching process runs. Any notification with a publication date older than the earliest publication date will NOT be matched.

Critical updates like errata, corrigenda or retraction notices are usually given new, recent publication dates by their publishers, and so should not be affected by this filter. However, if publishers do not adopt this convention, you risk not receiving such notices when their publication date is older than the maximum age.

Repository deposits are achieved using these mechanisms:

  • Router API - repository pulls from Router
  • SWORD2 protocol - Router pushes deposits to the repository.

API

The repository system will need your Router API-key: {{org_acc.api_key}}.

SWORD2

For SWORD2 deposits it is necessary to specify the repository configuration (which determines the XML structure) also the credentials for a SWORD enabled repository account and the SWORD collection URL.

These repository configurations are supported:

  1. Eprints Native - for basic Eprints implementation
  2. Eprints RIOXXplus v2 - for Eprints with the Router RIOXXplus plugin v2 installed *
  3. Eprints RIOXXplus v1 - for Eprints with the Router RIOXXplus plugin v1 installed *
  4. DSpace Native - for basic DSpace implementation
  5. DSpace RIOXX - for DSpace with the Router RIOXX patch installed.
    6. {#
  6. Router Native - for Esploro (& any other repositories) that can consume Router's native XML metadata structure.
    7. #}

Hybrid

Eprints RIOXXplus deposits are made using a hybrid approach where Router initially pushes article metadata via SWORD2, and the RIOXXplus plugin then pulls files (article PDF etc.) via the API *.

(*) Your API-key (see above) must be entered in the Eprints RIOXXplus plugin configuration file.
