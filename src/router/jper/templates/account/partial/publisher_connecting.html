Optimum performance

Files submitted by SFTP will be processed sooner if the SFTP client disconnects after a batch of files has been deposited. While an SFTP connection remains active, Router waits at least 30 minutes to process a file to avoid handling it while it is still being deposited.

SFTP deposit directories

When depositing notifications via SFTP, depending on whether you use Atypon zip format or not, you must use one of two possible target directories, /atypon_xfer/ or /xfer/ (both of which exist for your account). More information on supported zip file structures is available in our FTP Deposit Protocol document (section 3) and summarized below.

xfer directory for individual notification deposits

Individual (non-Atypon) notifications should each be deposited in either a separate zip file or a separate directory under the /xfer/ directory.  (As a minimum the zip file or directory must contain a JATS XML file describing the article, in addition the article PDF and related files such as images may be included.)

aytpon_xfer directory for Atypon zip file deposits

Notifications submitted as Atypon zip files must be deposited in the /atypon_xfer/ directory.  The Atypon zip file must contain one issue folder, and under that there may be one or more article folders, each of which has a JATS XML and any PDF files (at the top level), and any additional related files (which may be in sub-directories).
