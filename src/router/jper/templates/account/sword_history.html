SWORD deposit history{% if curr_org_is_super %}{{org_name}}{% endif %}

The table shows the history of deposits to a repository made using the SWORD protocol.

Total is the number of deposits made; Meta ... and Fulltext ... values indicate the number of successful or failed deposits of metadata and article files respectively (note that zero is replaced by hyphen for readability).

Error Messages lists all errors encountered.

• "Metadata error" indicates that no deposit record was created in the repository;

• "Content error" indicates that a metadata record was created, but the article file could not be attached to it. In this case the square brackets contain:
[ Link to article (zipfile or PDF) (notification ID) • Link to repository entry (entry ID) • DOI of the article concerned ].
You can use these links to manually download the article and attach it to your repository entry (NB. if you find the repository entry link doesn't work from your browser then can use the entry ID to find the record).

The last 10 days deposit history is shown by default.
